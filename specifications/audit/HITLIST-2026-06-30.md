**#20 — Whimsical error system (owner request, shipped on `feat/whimsical-error-system`).** Single source of truth at `backend/app/core/error_codes.py` (39 `QF-` codes; enum-`Severity` + frozen `ErrorCodeSpec` dataclass with `code`/`internal_description`/`whimsical_message`/`http_status`/`severity`/`notify_support`; import-time self-validation enforces the notify-allusion contract — notify codes allude to support being alerted, non-notify codes never claim it). The unified error envelope (`core/errors.py`) gains additive `code` + `whimsical` fields alongside the existing `errorCode`/`detail`/`traceId` (backward-compatible). Real failure paths wired to codes across `quiz.py`, `dependencies.py`, `results.py`, `topics.py`, and the background agent runner. Resend support-notify at `backend/app/services/support_notify.py` — thin `httpx` POST (no new dep), Redis `SET NX EX`-deduped (≤1 email/code/~15 min), fail-open (Redis fault/outage skips send — no email storm), fire-and-forget (never blocks/raises), NON-PII payload, graceful no-op + single WARNING when `RESEND_API_KEY` is absent (the key is added later). Frontend: one reusable `WhimsicalError` component (`components/common/`) renders the friendly message + the code as light-grey muted-token small text; wired into `ErrorPage`, `ErrorBoundary`, `GlobalErrorDisplay` (result errors), and the quiz error view; FE-only failures use a parallel `QF-FE-` map (`config/feErrorCodes.ts`).

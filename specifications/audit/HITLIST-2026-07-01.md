## ⏸ Reliability & scalability — DOCUMENTED, DEFER-TO-SCALE

Owner: defer until thousands of users, but record. All verified against live code; durable-jobs
Theme A and $-metered cost breaker are genuinely closed. Open (build-but-not-enabled) infra:

1. **Autoscale is advisory-only, never applied** — deploy action sets no min/max/scale-rule; bicep policy unconsumed. Effective min=1. Add a post-deploy `az containerapp update --scale-rule http-concurrency` step. `infrastructure/main.bicep`, `api-deploy.yml`.
2. **Non-durable LangGraph checkpointer + single worker** — Azure Redis lacks RedisJSON/RediSearch → silent InMemorySaver fallback; `WEB_CONCURRENCY=1`. Needs Redis Stack/Managed Redis Enterprise; then raise workers. Mitigated: `/next` rehydrates full state from Postgres.
3. **Cluster LLM concurrency cap ships `enabled=False`** — flip `llm.global_concurrency.enabled=true` when replicas/workers >1 (limiter already built, fail-open).
4. **Cross-topic character-image bleed (correctness, NOW-ish)** — reuse cache keyed on global `characters.name`, no topic scoping → shared archetype art leaks across topics. Widen to `(canonical_key/topic, name)`. Latent today (image gen flag OFF). `services/image_pipeline.py`.
5. Cold `/quiz/start` pins a worker up to ~120s — push character streaming to the durable background path.
6. **Cross-provider failover only real if BOTH keys provisioned** — GEMINI key IS in KV ✅; add a startup log asserting a distinct-provider fallback key exists.
7. No hourly/rate-of-change spend ceiling; single global daily $ counter is a DoS lever — add hourly sub-window + per-IP daily sub-cap + operator override.
8. Redis outage fails-open several cost/concurrency controls at once — documented deliberate availability trade-off.
9. LangGraph checkpointer no default TTL (latent until saver engages) — set default.
10. **`quiz_jobs` rows never reaped** — add a bounded prune in the recovery loop (small, safe, worth doing now).

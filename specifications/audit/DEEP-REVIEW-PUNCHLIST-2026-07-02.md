# Deep-Review Punch List — 2026-07-02

## Method

This punch list is the output of a deep adversarial review of the Quafel codebase run on **2026-07-02**, using **five adversarial lenses**, each hunting a different failure class:

1. **weak-code** — code that only appears to work: dead branches, silent contract mismatches, copy-paste divergence.
2. **ui-ux** — polish, motion, mobile, and design-system consistency defects on user-visible surfaces.
3. **security** — auth/config posture, cache poisoning, privacy correctness.
4. **branching** — hostile-timing and interrupted-flow traces: races, retries, refreshes, duplicate submits.
5. **test-gaps** — money-path and contract logic that can regress with the suite staying green.

Every non-trivial finding was then passed through **independent refuter verification** (up to 3 refutation attempts against the real code). Verdicts: `CONFIRMED` = survived all refutation passes with code-level reproduction; `unverified-low` = low-severity finding reported without a dedicated refuter pass. One finding was refuted during verification and is listed at the bottom for transparency.

All 35 surviving findings carry an estimated incremental Azure cost of **$0/month** to fix, so nothing was excluded from the main list on cost grounds (see the Deferred section).
